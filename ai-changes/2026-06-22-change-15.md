# AI Change Log — 2026-06-22-change-15

## Reason

Implement Phase 3 of v0.2: a settings window that lets users edit the dictation
configuration without manually editing `.env`. The window is integrated into the
system tray and persists changes back to the project `.env` file.

## Files changed

- `src/ui/settings_window.py` — new `SettingsWindow` form with fields for hotkey,
  push-to-talk, ASR model/language/device, LLM options, dry-run, and save/cancel
  buttons. Writes `Settings.model_dump()` to `.env` on save and emits
  `settings_saved`.
- `src/ui/tray.py` — `TrayIcon` now accepts an optional `settings_window`, enables
  the Settings action, and shows/raises the window when triggered.
- `src/ui/__init__.py` — exports `SettingsWindow`.
- `src/main.py` — creates `SettingsWindow`, passes it to `TrayIcon`, and connects
  `settings_saved` to update in-memory settings and show a restart advisory
  message for hotkey/ASR model changes.
- `tests/unit/test_settings_window.py` — new unit tests for loading settings,
  saving to `.env`, canceling, and `.env` serialization.
- `tests/unit/test_tray.py` — updated Settings action assertions and added tests
  for showing the settings window.
- `tests/unit/test_main.py` — updated mocks and assertions for `SettingsWindow`
  creation.
- `docs/architecture.md` — documented the new settings window and marked Phase 3
  complete.
- `docs/testing.md` — updated test counts and coverage table.

## Risk

- Some setting changes (hotkey, ASR model) require an application restart to take
  full effect. This is documented in the code and via the tray message on save.
- The `.env` writer quotes values with special characters; path values on Windows
  are quoted because they contain backslashes. This is compatible with
  pydantic-settings/python-dotenv.
