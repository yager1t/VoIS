# AI Change Log — 2026-06-22-change-19

## Reason

Phase 2 of v0.3 adds the text correction layer. Raw ASR transcripts often misrecognize
specialized terms, names, and abbreviations. `TextCorrector` applies vocabulary
replacements from `VocabularyManager` before post-processing, using longest-match
ordering and word-boundary checks to avoid corrupting ordinary words.

## Files changed

- `src/dictionary/corrector.py` — new `TextCorrector` class and `_preserve_case` helper.
- `src/dictionary/__init__.py` — exports `TextCorrector`.
- `src/app.py` — initializes `VocabularyManager` and `TextCorrector`; calls corrector
  in `stop_recording()` when `dictionary_enabled` is true.
- `tests/unit/test_corrector.py` — unit tests for replacement, word-boundary protection,
  longest-match priority, case preservation, empty text, no-op terms, and multiple
  replacements.
- `tests/unit/test_app.py` — mocks dictionary/corrector and adds tests for enabled/disabled
  correction paths.
- `docs/architecture.md` — documents the corrector in the data flow and module breakdown,
  and updates the v0.3 roadmap.

## Risk

- Low-to-moderate. Corrector only runs when `dictionary_enabled` is true (default).
- Word-boundary checks prevent replacing substrings inside larger words, but multi-word
  phrases with unusual punctuation may need future refinement.
- Longest-match ordering prevents shorter terms from shadowing longer phrases.
