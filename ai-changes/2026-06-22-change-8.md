# AI Change Log — 2026-06-22-change-8

## Reason
Implement Phase 2 of the test plan: add unit tests for previously uncovered modules
(`AudioCapture`, `logging_config`, `main`) and expand coverage for existing modules
with edge cases, validation, and error paths. Also add a bounded memory limit to
`AudioBuffer` and refactor `ModelManager` so unit tests can patch model download /
instantiation without importing heavy ASR packages.

## Files changed

### Source changes
- `src/app.py` — added `_prepare_audio`, `toggle_recording`, bounded buffer usage,
  non-busy shutdown event handling.
- `src/asr/model_manager.py` — extracted `_download_model` and `_create_whisper_model`
  wrapper functions so tests can patch them without touching faster-whisper.
- `src/audio/buffer.py` — added optional `max_seconds` / `sample_rate` bound to
  prevent unbounded memory growth during long recordings.
- `src/audio/vad.py` — made `webrtcvad` import lazy/optional, added
  `split_on_silence` context-keeping logic.
- `src/config.py` — added `audio_max_record_seconds` setting.
- `AI_DEV_RULES.md` — simplified and cross-referenced `docs/ai_working_guide.md`.
- `README.md`, `docs/testing.md`, `pyproject.toml`, `requirements-dev.txt`,
  `scripts/run_tests.bat`, `scripts/run_tests.sh` — minor updates for markers and
  timeout support.

### Test changes
- `tests/unit/test_capture.py` — new tests for `AudioCapture`.
- `tests/unit/test_logging_config.py` — new tests for logging setup.
- `tests/unit/test_main.py` — new tests for CLI parsing and overrides.
- `tests/unit/test_app.py` — updated for toggle mode and VAD integration.
- `tests/unit/test_asr_provider.py` — added model manager wrapper tests.
- `tests/unit/test_buffer.py` — added bounded buffer test.
- `tests/unit/test_config.py` — added validation tests for new fields.
- `tests/unit/test_model_manager.py` — added wrapper patch tests.
- `tests/unit/test_vad.py` — added `split_on_silence`, short audio, and invalid
  sample rate tests.

### Documentation
- `docs/ai_working_guide.md` — new file with practical safety rules for AI
  assistants (command scope, ASR/model patching, runtime risk areas).

## Diff summary
~501 insertions, ~153 deletions across 18 files.

## Risks
- `AudioBuffer` now drops oldest samples when `max_seconds` is set; callers must
  ensure this matches expected maximum recording duration.
- Lazy `webrtcvad` import changes failure mode from import-time to construction-time.
- Tests now depend on `pytest-timeout`.
