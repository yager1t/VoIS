# AI Change Log — 2026-06-23-change-4

## Reason

Fix two issues reported in the v0.6.0 portable Windows build:
1. `voice-to-cursor.exe` (windowed) crashed on startup because loguru tried to log to `sys.stderr`, which is `None` in a PyInstaller windowed executable.
2. When the user had CUDA installed but the portable bundle was CPU-only, `faster-whisper` failed with `RuntimeError: Library cublas64_12.dll is not found or cannot be loaded`.

## Files changed

- `src/logging_config.py` — skip the console sink when `sys.stderr` is `None`.
- `src/asr/whisper_provider.py` — fall back to `device="cpu"` / `compute_type="int8"` when model loading fails with a CUDA/cuBLAS/cuDNN error.
- `tests/unit/test_logging_config.py` — added tests for console sink behavior.
- `tests/unit/test_asr_provider.py` — added tests for CUDA fallback and non-CUDA error propagation.
