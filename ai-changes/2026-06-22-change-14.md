# AI Change Log — 2026-06-22-change-14

## Reason
Implement Phase 2 of v0.2: add a PyQt6 system tray icon so the application no longer
blocks a terminal window. `App` keeps its existing hotkey/audio loop and is moved to a
background `QThread`; the main thread owns `QApplication` and the tray UI.

## Files changed
- `pyproject.toml` — added `PyQt6>=6.6` to project dependencies.
- `requirements.txt` — added `PyQt6>=6.6`.
- `src/app.py` — added `recording_started`, `recording_stopped`, and `text_injected`
  callback attributes; added `is_running()` and `_invoke_callback()` helpers.
- `src/main.py` — refactored entry point to create `QApplication`, `TrayIcon`, and a
  `_Worker(QObject)` running `App` in a `QThread`.
- `src/ui/tray.py` — new `TrayIcon` class with context menu, Start/Stop toggle,
  disabled Settings placeholder, Exit, balloon notifications, and recording indicator.
- `src/ui/__init__.py` — exports `TrayIcon`.
- `tests/unit/test_app.py` — added callback invocation tests.
- `tests/unit/test_main.py` — updated to mock Qt objects and assert tray/worker setup.
- `tests/unit/test_tray.py` — new unit tests for `TrayIcon` using fake Qt classes.
- `docs/architecture.md` — documented Qt tray integration and threading model.
- `docs/testing.md` — updated test counts and coverage table.

## Risk
- PyQt6 adds a large dependency; CI install time on Windows may increase slightly.
- Callbacks from the worker thread invoke tray Qt methods; real-world thread safety
  may need queued signal/slot migration in a future phase.
- `src/ui/tray.py` tests rely on `sys.modules` patching and forced module reload to
  avoid loading real PyQt6 classes in the test process.
