# Change 18: Dictionary Storage and Vocabulary Manager (v0.3 Phase 1)

**Date:** 2026-06-22
**Change-ID:** 20260622-kimi-004604
**AI:** yes
**Model:** kimi

## Reason

Implement Phase 1 of v0.3: a persistent dictionary system and vocabulary manager
that supports static, context-specific, and user-defined terms. This enables
context-aware dictation (general, chat, email, code) and user customization.

## Changed files

- `src/config.py` — added `context_mode`, `dictionary_enabled`,
  `dictionary_learning_enabled`, and `vocab_dir`; updated `ensure_dirs()`.
- `src/dictionary/__init__.py` — public exports.
- `src/dictionary/base.py` — `VocabularySource`, `ContextMode`, `DictionaryEntry`.
- `src/dictionary/context_modes.py` — context prompts, starter vocabulary, parser.
- `src/dictionary/storage.py` — `VocabularyStorage` JSON persistence.
- `src/dictionary/vocab_manager.py` — `VocabularyManager` with override priority.
- `data/vocab/static.json` — empty default static vocabulary.
- `data/vocab/context_chat.json` — casual chat fillers.
- `data/vocab/context_email.json` — formal email phrases.
- `data/vocab/context_code.json` — common programming terms.
- `.gitignore` — allow `data/vocab/` to be tracked while ignoring runtime data.
- `tests/unit/test_dictionary_storage.py` — storage load/save and defaults.
- `tests/unit/test_vocab_manager.py` — manager behavior and override priority.
- `tests/unit/test_context_modes.py` — context helpers.
- `tests/unit/test_config.py` — new setting defaults and env loading.

- Default vocabulary files are small and read-only after creation.
- No runtime migration needed because files are created on first load.
