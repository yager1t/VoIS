# AI Change Log — 2026-06-22-change-2

## Reason

Implement Phase 2 (Audio Capture + VAD) of the Voice-to-Cursor AI Dictation System.
This change adds the audio capture layer (`AudioCapture`), thread-safe buffering
(`AudioBuffer`), and a pluggable voice-activity-detection interface with a WebRTC VAD
implementation (`WebRTCVADProvider`). It also wires placeholder recording methods into
the `App` class and adds unit tests plus an optional smoke-test script.

## Files Changed

- `pyproject.toml` — added audio/VAD dependencies and bumped version to `0.2.0`; updated `mypy` target to Python 3.12 to avoid a type-statement compatibility error with `numpy` 2.5 stubs.
- `requirements.txt` — added `sounddevice>=0.4`, `numpy>=1.24`, `soundfile>=0.12`, `webrtcvad>=2.0`.
- `requirements-dev.txt` — inherited updated runtime requirements.
- `src/app.py` — `App` now creates an `AudioCapture` instance and exposes `start_recording()` / `stop_recording()` stubs.
- `src/audio/__init__.py` — exports `AudioBuffer`, `AudioCapture`, `VADProvider`, `WebRTCVADProvider`.
- `src/audio/buffer.py` — new `AudioBuffer` class: append, get, clear, duration, and trailing-window retrieval; thread-safe.
- `src/audio/capture.py` — new `AudioCapture` class using `sounddevice`; supports push-to-talk and toggle modes, callback registration, and graceful stream lifecycle handling.
- `src/audio/vad.py` — new `VADProvider` interface and `WebRTCVADProvider`; float32→int16 conversion, framing helpers, and `split_on_silence` segmentation.
- `tests/unit/test_buffer.py` — unit tests for `AudioBuffer`.
- `tests/unit/test_vad.py` — unit tests for VAD utilities and a mocked `WebRTCVADProvider`.
- `scripts/test_audio.py` — optional smoke-test script that records audio, saves it to `data/debug_record.wav`, and reports a VAD speech ratio.

## Risks

- `webrtcvad` does not provide pre-built Windows wheels for Python 3.12; building from source requires Microsoft Visual C++ 14.0. On Windows it may be easier to use the drop-in `webrtcvad-wheels` package in development environments.
- `sounddevice` depends on PortAudio; on some systems the default input device may not be available, causing `AudioCapture.start()` to raise.
- The `App.start_recording()` / `stop_recording()` methods are stubs; real hotkey integration will happen in Phase 4.
- `mypy` target was moved from Python 3.11 to 3.12 because `numpy` 2.5 stubs use Python 3.12 `type` statement syntax. Projects that must be type-checked against 3.11 should pin `numpy<2.5`.
