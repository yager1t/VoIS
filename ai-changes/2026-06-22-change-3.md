# AI Change Log — 2026-06-22-change-3

## Reason

Implement Phase 3 (ASR Layer) of the Voice-to-Cursor AI Dictation System:
- Add an abstract `ASRProvider` interface and a concrete `FasterWhisperProvider`.
- Introduce `ModelManager` for local model path resolution and download caching.
- Wire ASR into `App.transcribe_audio()`.
- Switch the VAD dependency to a Windows-compatible wheel (`webrtcvad-wheels`).
- Add faster-whisper / torch / huggingface-hub dependencies.
- Provide unit tests and an optional smoke script.

Key implementation details:
- `FasterWhisperProvider` loads the model lazily and writes audio to a temporary WAV before transcription.
- `transcribe_streaming` currently returns a final result; partial streaming is planned for v0.4.
- `ModelManager.ensure_model` downloads models into `models/<model_name>` only when missing.
- `App.asr` is lazily initialized so importing `App` does not trigger model loading.

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| `torch` / `faster-whisper` install size | Large downloads on first install | Document CPU-only install; use `https://download.pytorch.org/whl/cpu`. |
| First model download | Requires internet and disk space | Default model is `base`; manager caches under `models/` (gitignored). |
| Temporary WAV file cleanup | Disk leak on crash | Wrapped in `try/finally` with `unlink(missing_ok=True)`. |
| Windows-specific `webrtcvad-wheels` | Linux/macOS wheel availability | Fallback comment retained; package is a drop-in for the original `webrtcvad`. |
| Streaming is not partial | Higher perceived latency | Marked as v0.4 improvement; current behavior is final transcription per chunk. |
