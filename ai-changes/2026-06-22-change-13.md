# AI Change Log — 2026-06-22-change-13

## Reason

Implemented Phase 1 of v0.2: the LLM Post-processing Layer for the Voice-to-Cursor AI Dictation System.
The MVP pipeline ended at ASR → text injection. This change introduces an optional post-processing step that can improve grammar, punctuation, and formatting via a local Ollama LLM, while keeping a deterministic fallback formatter when the LLM is disabled or unavailable.

## Changed files

### New
- `src/postprocess/base.py` — `PostProcessor` abstract interface.
- `src/postprocess/formatter.py` — `TextFormatter` deterministic cleanup.
- `src/postprocess/llm_client.py` — `OllamaClient` and `LLMPostProcessor` with graceful fallback.
- `src/postprocess/__init__.py` — exports and `create_post_processor(settings)` factory.
- `tests/unit/test_postprocess_formatter.py` — deterministic formatter tests.
- `tests/unit/test_ollama_client.py` — mocked HTTP success/fallback tests.
- `tests/unit/test_postprocess_factory.py` — factory selection tests.

### Modified
- `pyproject.toml` — added `httpx>=0.27` dependency.
- `requirements.txt` — added `httpx>=0.27` dependency.
- `src/config.py` — added `llm_model`, `llm_timeout`, `llm_prompt` fields under LLM section.
- `src/app.py` — wired `post_processor` into `App`; calls `process()` after transcription.
- `tests/unit/test_app.py` — updated assertions to expect post-processed text.
- `tests/unit/test_config.py` — added assertions for new LLM fields.
- `tests/integration/test_end_to_end.py` — updated expected injected text to reflect formatting.
- `docs/architecture.md` — documented the post-processing layer and module breakdown.
- `docs/testing.md` — updated coverage table and test counts.

- LLM calls add latency; mitigated by configurable timeout (default 5 s) and fallback to raw text on any error.
- New `httpx` dependency must be available in CI; already added to `pyproject.toml` and `requirements.txt`.
- Integration tests now expect formatted output; updated accordingly.
