## Reason

Phase 5 of v0.4 adds a latency benchmark harness that measures and compares the
transcription pipeline in streaming (incremental) and batch (full-file) modes.
The harness must be runnable in CI/unit runs without a real model, while still
supporting real-model benchmarking via explicit CLI flags and pytest markers.

## Files changed

- `src/benchmarks/__init__.py` ŌĆö new benchmarks package marker.
- `src/benchmarks/latency.py` ŌĆö `LatencyResult` dataclass, `LatencyBenchmark`
  class, `AlwaysSpeechVAD`, and `MockASRProvider`. Streaming mode reuses
  `StreamingTranscriber` chunk-by-chunk; batch mode calls `asr.transcribe()`
  once. Records `time_to_first_partial`, `time_to_final`,
  `total_audio_seconds`, `rtf`, and `transcript`.
- `benchmarks/run_latency.py` ŌĆö top-level CLI that generates/loads audio,
  accepts `--mode streaming|batch|both`, `--real-model`, and model-related
  flags. Without `--real-model` it uses the mock provider and prints a JSON
  summary.
- `tests/unit/benchmarks/test_latency.py` ŌĆö unit tests for both benchmark
  modes, result fields, and sensible RTF. Includes a real-model test marked
  `@pytest.mark.slow` and `@pytest.mark.requires_model`.
- `tests/unit/benchmarks/test_run_latency.py` ŌĆö unit tests for the CLI script
  and helper functions so coverage stays above the 80% threshold.
- `tests/unit/benchmarks/__init__.py` ŌĆö new test package marker.
- `pyproject.toml` ŌĆö added `benchmarks` to the `[tool.coverage.run]` source
  list so the top-level benchmark script is included in coverage reports.
