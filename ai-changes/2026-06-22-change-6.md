# AI Change Log — 2026-06-22 Change 6

## Reason

Phase 6 of the Voice-to-Cursor AI Dictation System integrates the previously
implemented components (audio capture, VAD, ASR, hotkey, text injection) into a
working end-to-end pipeline and adds a smoke-test harness. The previous
`src/app.py` main loop was a busy `while self._running: pass` that consumed
100% CPU, and VAD was not yet used to trim silence before transcription.

## Changes

### 1. Fixed main loop in `src/app.py`
- Replaced busy loop with `self._shutdown_event.wait(timeout=0.1)` using a
  `threading.Event`.
- `stop()` now calls `self._shutdown_event.set()`.

### 2. Integrated VAD into the pipeline
- `App.__init__` now creates a `WebRTCVADProvider` configured from settings.
- Added module-level `trim_silence()` helper and `_prepare_audio()` method.
- `stop_recording()` trims leading/trailing silence and skips ASR/injection
  when no speech is detected.

### 3. Improved CLI in `src/main.py`
- Added `--model`, `--language`, `--device`, `--hotkey`, `--toggle`, and
  `--dry-run` arguments.
- Added `apply_cli_overrides()` to mutate `Settings` before constructing `App`.

### 4. Added dry-run mode
- Added `dry_run: bool = False` to `Settings`.
- `App.inject_text()` logs text instead of injecting when `dry_run` is enabled.

### 5. Updated `src/config.py`
- Added `vad_aggressiveness: int = 1`.
- Added `vad_trim_seconds: float = 0.3`.
- Added `dry_run: bool = False`.

### 6. Tests
- Created `tests/unit/test_app.py` covering:
  - `start_recording` clears buffer and starts capture.
  - `stop_recording` stops capture, calls ASR, and injects text.
  - `dry_run` skips injection.
  - Empty audio and no-speech-after-VAD skip ASR/injection.
  - `stop()` stops hotkey and sets shutdown event.
  - Main loop exits after `stop()` is called.
  - `trim_silence` behavior.

### 7. Smoke test script
- Created `scripts/smoke_test.py` that runs the full app for a limited time.
- Defaults to dry-run mode; supports `--no-dry-run`.
- Auto-stops after `--duration` seconds or on Ctrl+C.

### 8. Updated `README.md`
- Added Quick start section.
- Documented dry-run and real modes.
- Documented CLI flags and smoke test script.
- Added Windows-only MVP note.

## Diff summary

```
src/app.py          | non-busy main loop, VAD integration, dry-run support
src/main.py         | new CLI flags and override application
src/config.py       | vad_aggressiveness, vad_trim_seconds, dry_run
tests/unit/test_app.py | new unit tests for App lifecycle and VAD
scripts/smoke_test.py  | end-to-end smoke test harness
README.md           | quick start, CLI reference, platform note
```

## Risk

- Low-to-moderate. Changes are confined to the application orchestrator and CLI.
- VAD trimming could drop very quiet or truncated speech if
  `vad_trim_seconds`/`vad_aggressiveness` are too aggressive; defaults are
  conservative.
- The Windows-only text injector remains unchanged; non-Windows platforms are
  still unsupported.
