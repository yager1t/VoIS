# AI Change Log: Phase 3 Integration Tests

**Change-ID:** 20260622-kimi-de3918  
**Date:** 2026-06-22  
**Model:** kimi  
**Scope:** test

## Reason

Implement Phase 3 of the approved test plan: integration tests for the
Voice-to-Cursor audio pipeline and end-to-end flow. The goal is to verify that
the composed components (audio capture, buffer, VAD, ASR, injection) work
together while still avoiding real hardware, hotkeys, text injection, and ASR
model downloads.

## Changed files

- `tests/conftest.py` (new)  
  Shared pytest fixtures: isolated `Settings`, sample-rate, synthetic speech /
  silence / mixed audio, mock Whisper model, and patched model manager wrappers.
- `tests/integration/test_pipeline.py` (new)  
  Three `@pytest.mark.integration` tests covering audio buffer → VAD,
  simulated `sd.InputStream` callback accumulation, and ASR pipeline with
  patched `src.asr.model_manager` wrappers.
- `tests/integration/test_end_to_end.py` (new)  
  Four `@pytest.mark.integration` tests covering push-to-talk flow, toggle-mode
  flow, dry-run behavior, and VAD-silence short-circuit using fully mocked
  `HotkeyManager`, `AudioCapture`, `TextInjector`, and ASR.
- `docs/testing.md`  
  Added "Running integration tests" section and updated test count to 112 unit
  + 7 integration tests.

## Risk

- Low. All new code lives under `tests/` and `docs/`. No production behavior was
  changed.
- Integration tests mock platform-specific dependencies (`pynput`, `sounddevice`
  streams, `ctypes`/`SendInput`) and patch only the approved wrappers in
  `src.asr.model_manager`.
