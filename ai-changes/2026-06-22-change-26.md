# AI Change Log: Phase 3 v0.4 — Background Final Transcription

**Change-ID:** 20260622-kimi-6f8a1b
**Commit:** 310d8eb
**Date:** 2026-06-22
**Model:** kimi
**Scope:** feature

## Reason

Implement Phase 3 of v0.4: after a streaming recording stops, inject the fast
streaming transcript immediately and then run a full, more accurate
transcription on the complete recorded audio in a background thread. The final
result is emitted via a new `text_finalized` callback so the UI can optionally
replace the streaming text later. When streaming is disabled, the existing
blocking transcription behavior remains unchanged.

## Changed files

- `src/asr/final_transcriber.py` (new)
  - Added `FinalTranscriber` helper class.
  - Runs `ASRProvider.transcribe()` on a daemon thread and reports the result
    through a callback.
  - Supports `stop()` cancellation and thread-safe start/stop state.
- `src/config.py`
  - Added `final_transcription_enabled: bool = True`
  - Added `replace_streaming_with_final: bool = False` (reserved for UI)
- `src/app.py`
  - Added `text_finalized: Callable[[str], None] | None` callback.
  - Added `_finalize_text()` helper to apply dictionary correction and
    post-processing.
  - Added `_on_final_transcription()` callback used by `FinalTranscriber`.
  - In `stop_recording()`, when streaming is enabled, inject the streaming
    transcript immediately and then start a `FinalTranscriber` on the prepared
    audio (when `final_transcription_enabled` is true).
  - In `start_recording()`, cancel any pending final transcription from a
    previous session.
  - Non-streaming mode keeps the existing synchronous transcription path.
- `tests/unit/asr/test_final_transcriber.py` (new)
  - Tests callback invocation, empty-result handling, cancellation before
    start, stop idempotency, start idempotency, and ASR-error handling.
- `tests/unit/test_app.py`
  - Updated `_build_app` fixture to patch `FinalTranscriber` and expose the
    created mock instances.
  - Added tests verifying `text_finalized` is emitted when streaming is
    enabled, skipped when `final_transcription_enabled=False`, and that the
    non-streaming synchronous path remains unchanged.
  - Added test verifying `start_recording()` cancels a pending final
    transcriber.

## Risks

- `FinalTranscriber` runs real ASR inference in a background thread. The ASR
  `transcribe()` call itself is not interruptible; cancellation only skips the
  callback if stop is requested before or after transcription.
- The final callback receives corrected and post-processed text, which may
  differ from the initially injected streaming text.
- `replace_streaming_with_final` is added to settings but not wired into any
  replacement behavior; the UI must implement optional replacement itself.
