# AI Change Log — 2026-06-22-change-20

## Reason

Phase 3 of v0.3 adds ASR biasing with dictionary hotwords and an initial prompt.
Specialized terms are often misrecognized by Whisper; supplying context-specific
prompt text and a hotword list helps nudge the model toward the user's vocabulary
before the text-correction layer runs.

## Files changed

- `src/dictionary/bias.py` — new `ASRBias` class with `initial_prompt()`, `hotwords()`,
  and `_truncate()` helpers. Uses context description + vocabulary terms, limited to
  250 characters and 30 hotwords, sorted by count then alphabetically.
- `src/dictionary/__init__.py` — exports `ASRBias`.
- `src/asr/whisper_provider.py` — lazily initializes `ASRBias` from settings and passes
  `initial_prompt` and `hotwords` to `WhisperModel.transcribe()` only when
  `dictionary_enabled` is true and the values are non-empty.
- `tests/unit/test_asr_bias.py` — new unit tests for prompt construction, hotword
  limiting/sorting, disabled dictionary behavior, and truncation.
- `tests/unit/test_asr_provider.py` — adds tests verifying that bias hints are passed
  to the model when the dictionary is enabled and omitted when disabled.
- `docs/architecture.md` — documents `ASRBias` in the data flow and module breakdown,
  and marks Phase 3 of v0.3 complete.

## Risk

- Low. Biasing only runs when `dictionary_enabled` is true (default).
- Prompt and hotword limits prevent overloading the ASR model.
- The feature depends on the existing `VocabularyManager`; any future change to its
  internal storage layout must also update `ASRBias.hotwords()`.
