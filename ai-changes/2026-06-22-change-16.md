# Change log: Phase 4 - Recording indicator and tray notifications

**Date:** 2026-06-22
**Change-ID:** 20260622-kimi-c0c14a
**AI:** yes
**Model:** kimi

## Reason

Phase 4 of v0.2 adds a recording-state indicator and short tray notifications so
users get clear visual feedback while dictating. The indicator must be safe to
trigger from `App`'s worker thread, so all Qt tray updates are marshalled through
Qt signals/slots.

## Changed files

- `src/ui/tray.py`
  - Added `recording_state_changed` and `show_notification` Qt signals.
  - Added public thread-safe methods `set_recording(recording)` and
    `notify(title, message)` that emit signals.
  - Signal slots (`_on_recording_changed`, `_on_show_notification`) perform the
    actual icon/message updates on the Qt main thread.
  - Removed automatic `App` callback wiring from `__init__`; wiring is now done
    explicitly in `main()`.

- `src/main.py`
  - Wired `app.recording_started`, `app.recording_stopped`, and
    `app.text_injected` to tray methods after creating `TrayIcon`.
  - Recording start shows the recording icon and a "Recording..." balloon.
  - Recording stop restores the idle icon.
  - `text_injected` shows "Transcribed: <preview>" in dry-run mode and
    "Text injected" in normal mode.

- `tests/unit/test_tray.py`
  - Added fake `pyqtSignal` and updated mocked Qt modules.
  - Replaced the old callback-wiring test with signal-connection tests.
  - Added tests for `set_recording()` and `notify()` signal emission and side
    effects.

- `tests/unit/test_main.py`
  - Added tests verifying `App` recording callbacks are wired to tray methods.
  - Added tests for dry-run vs normal-mode text-injected notifications.

- `docs/architecture.md`
  - Marked Phase 4 (recording indicator and notifications) as completed.
  - Updated the `tray.py` description to mention thread-safe signal-based
    updates.

- `docs/testing.md`
  - Updated coverage table and test counts.

## Risk

- The default signal connection type is `AutoConnection`, which Qt promotes to a
  queued connection when the sender lives in a different thread than the
  receiver. This should keep tray updates thread-safe, but real multi-threaded
  behavior should be verified in a manual smoke run before release.
- Tray balloon notifications may be silently suppressed by Windows Focus Assist
  or other OS settings; this is normal system behavior.
