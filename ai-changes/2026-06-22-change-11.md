# AI Change Log: Smoke/E2E Tests and CI

**Change-ID:** 20260622-kimi-11smoke  
**Date:** 2026-06-22  
**Model:** kimi  
**Scope:** test infrastructure, CI, documentation

## Reason

Implement Phase 4 of the approved test plan: add a pytest-based smoke test
harness that is skipped by default, a `--run-smoke` custom pytest option, helper
scripts, a GitHub Actions CI workflow, and update project documentation.

## Changed files

- `tests/smoke/__init__.py` — new smoke test package marker.
- `tests/smoke/test_smoke.py` — new smoke test exercising the full `App`
  pipeline in `dry_run=True` mode with synthetic audio; stubs real microphone,
  hotkey, injection, VAD, and ASR dependencies.
- `tests/conftest.py` — appended `pytest_addoption`, `pytest_configure`, and
  `pytest_collection_modifyitems` hooks for `--run-smoke`.
- `pyproject.toml` — removed `not smoke` from default `addopts` so smoke tests
  are collected and explicitly skipped by conftest when `--run-smoke` is absent.
- `scripts/run_smoke.sh` — new bash helper to run smoke tests with
  `--run-smoke --timeout=30`.
- `scripts/run_smoke.bat` — new Windows batch helper to run smoke tests.
- `scripts/smoke_test.py` — added deprecation note pointing to the new harness.
- `.github/workflows/ci.yml` — new GitHub Actions workflow for Python 3.11 and
  3.12 on `windows-latest` running lint, type check, and unit/integration tests
  with coverage.
- `docs/testing.md` — added "Running smoke tests" and "CI" sections; updated
  test counts.
- `README.md` — added CI badge placeholder and updated test count.

## Validation run

Smoke tests were not executed with `--run-smoke` per safety rules.

## Risk

- The smoke test stubs all platform dependencies, so it validates orchestration
  more than real hardware/OS integration. This is intentional for a safe
  developer-machine run; true hardware smoke testing still requires manual
  `scripts/smoke_test.py` or a controlled environment.
- The CI badge URL uses `Jury/Vo-IS`; verify this matches the actual GitHub
  repository slug after enabling Actions.
