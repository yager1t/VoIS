# Change 3 — Add Windows installer build

## Summary

Created a PyInstaller-based Windows build pipeline for Voice-to-Cursor, producing both a portable zip archive and an Inno Setup `.exe` installer. The build uses `--onedir`, ships a windowed GUI executable plus a console executable for dry-run/debugging, bundles the `.env.template` and `data/vocab` skeleton, and avoids CUDA bloat because the project uses CPU-only torch.

## Files changed

- `.env.template` (added) — configuration template copied into the distribution.
- `installer/entrypoint.py` (added) — PyInstaller entry-point wrapper that calls `src.main.main()`.
- `installer/voice-to-cursor.spec` (added) — PyInstaller spec building two shared onedir executables.
- `installer/hooks/hook-webrtcvad.py` (added) — custom hook overriding the upstream webrtcvad metadata hook that fails for `webrtcvad-wheels`.
- `installer/voice-to-cursor.iss` (added) — Inno Setup installer script.
- `scripts/build_installer.py` (added) — one-command build script that runs PyInstaller, creates the portable zip, and compiles the Inno Setup installer.

## Artifacts

- `dist/voice-to-cursor/` — PyInstaller onedir output (windowed + console executables, `_internal` dependencies).
- `dist/installer/Voice-to-Cursor-0.6.0-portable.zip` — portable archive (~285.6 MB).
- `dist/installer/Voice-to-Cursor-0.6.0-Setup.exe` — Inno Setup installer (~208.4 MB).

## Testing

- Dry-run test: launched `dist/voice-to-cursor/voice-to-cursor-console.exe --dry-run`; the app loaded settings, started the hotkey listener, and entered the main loop without crashing. The process was terminated after ~10 s. No microphone capture or text injection was performed.
- No Whisper model was downloaded during the build.
