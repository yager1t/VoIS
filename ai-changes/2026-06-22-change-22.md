# AI Change Log — 2026-06-22-change-22

## Reason
Phase 5 of v0.3 adds dictionary and learning controls to the UI: context mode
selection, dictionary/learning toggles, a vocabulary editor dialog, and a tray
"Add correction..." action. These expose the existing dictionary backend to the
user without changing the dictation pipeline.

## Files changed
- `src/ui/settings_window.py` — added context mode dropdown, dictionary and
  learning checkboxes, and an "Open vocabulary editor" button. Added
  `set_vocab_editor()` and wired save to persist the new fields.
- `src/ui/vocab_editor.py` — new `VocabularyEditor` dialog for viewing,
  adding, editing, and removing user vocabulary entries.
- `src/ui/tray.py` — added "Add correction..." action that prompts for
  original/corrected text and calls `app.record_correction()`.
- `src/ui/__init__.py` — exported `VocabularyEditor`.
- `src/main.py` — created `VocabularyEditor` and linked it to the settings
  window.
- `src/dictionary/vocab_manager.py` — added `get_all_entries()`.
- `docs/architecture.md` — documented the dictionary UI layer and marked
  Phase 5 of v0.3 complete.
- `tests/unit/test_vocab_editor.py` — new mocked tests for the editor.
- `tests/unit/test_settings_window.py` — tests for new controls and save output.
- `tests/unit/test_tray.py` — tests for the correction action.
- `tests/unit/test_main.py` — tests for editor creation/linkage.
- `tests/unit/test_vocab_manager.py` — test for `get_all_entries()`.
