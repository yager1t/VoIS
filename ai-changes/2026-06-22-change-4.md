# AI Change Log — 2026-06-22-change-4

## Change-ID
20260622-kimi-16a0d1

## Reason
Phase 4 of the Voice-to-Cursor MVP: add a global hotkey manager so the user can
start/stop voice recording with a system-wide key (default `f9`) or a
combination such as `<ctrl>+f9`. This wires the hotkey layer into `App` and
implements push-to-talk and toggle modes.

## Scope
- Added `pynput>=1.7` dependency to `pyproject.toml` and `requirements.txt`.
- Created `src/hotkey/` package:
  - `base.py`: abstract `HotkeyManager` and `parse_hotkey()` helper.
  - `windows.py`: `PynputHotkeyManager` using `pynput` with daemon listener.
  - `__init__.py`: exports and `create_hotkey_manager()` factory.
- Updated `src/app.py` to own a `HotkeyManager`, wire press/release to
  `start_recording()` / `stop_recording()`, and start/stop the listener with the
  app lifecycle.
- Added `tests/unit/test_hotkey.py` covering parsing, lifecycle, push-to-talk,
  toggle, and non-repeating press behavior (with mocked `pynput` listener).
- Added `scripts/test_hotkey.py` 10-second F9 smoke script.

## Risk
- `pynput` requires permissions for global hooks on some systems; on Windows it
  runs from a normal user account but may be flagged by aggressive AV.
- Push-to-talk release is tied to the main key release; if modifier keys are
  released first, behavior is still correct because we only check main key.
- `__del__` cleanup is best-effort; explicit `stop()` is preferred.
