# AI Change Log — 2026-06-22-change-5

## Change-ID
20260622-kimi-ab49b9

## Reason
Phase 5 of the Voice-to-Cursor MVP: add a text injection layer so that
transcribed speech can be inserted at the current cursor position in the active
Windows application. This implements the Windows-specific injector using
``SendInput`` with Unicode events and provides a clipboard-based ``Ctrl+V``
fallback for applications that do not accept direct Unicode injection.

## Scope
- Added ``pyperclip>=1.8`` dependency to ``pyproject.toml`` and
  ``requirements.txt``.
- Created ``src/injection/`` package:
  - ``base.py``: abstract ``TextInjector`` interface.
  - ``windows.py``: ``WindowsTextInjector`` using ``ctypes`` + ``SendInput``
    with ``KEYEVENTF_UNICODE``, plus clipboard fallback.
  - ``__init__.py``: exports and ``create_text_injector()`` factory.
- Updated ``src/config.py`` with ``injection_delay_ms`` and
  ``injection_fallback_to_clipboard`` settings.
- Updated ``src/app.py`` to own a ``TextInjector``, wire an ``AudioBuffer`` to
  ``AudioCapture``, and implement ``inject_text()``. ``stop_recording()`` now
  transcribes buffered audio and logs/injects the result (pipeline placeholder
  for Phase 6).
- Added ``tests/unit/test_injection.py`` covering direct Unicode injection,
  per-character delay, clipboard fallback, factory behavior, and platform
  availability.
- Added ``scripts/test_injection.py`` 3-second smoke script that injects a
  mixed English/Cyrillic string.

## Risk
- ``SendInput`` requires a normal user session; it may not work in elevated or
  certain secure input fields.
- Some applications (e.g. older terminals, games) may ignore Unicode input;
  the clipboard fallback mitigates this but overwrites the user's clipboard.
- Clipboard fallback simulates ``Ctrl+V``; if the active application overrides
  this shortcut the paste will not occur.
